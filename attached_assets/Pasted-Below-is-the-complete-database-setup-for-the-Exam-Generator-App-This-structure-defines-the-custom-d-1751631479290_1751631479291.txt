Below is the complete database setup for the Exam Generator App. This structure defines the custom data types (tables) along with their key fields and relationships. You can implement this using Bubble’s custom data types and fields. Adjust and extend as needed during development.

1. User & Access Management
User
Purpose: Represents all users (teachers, students, administrators) and stores profile and role data.

Key Fields:

Name: (text) – Full name.

Email: (text, unique) – User email address.

Password: (text, encrypted) – Login credential.

Role: (Option set) – e.g., “Teacher”, “Student”, “Admin”.

Profile Image: (image) – Optional profile picture.

Associated Testbanks: (list of Testbank) – Testbanks managed by a teacher.

Created Quizzes: (list of Quiz) – Quizzes/exams authored.

Assigned Quizzes: (list of Quiz) – Exams assigned to the user (for students).

Notifications: (list of Notification) – Alerts and messages.

2. Content & Assessment Creation
Testbank
Purpose: A collection of questions organized by themes, objectives, or taxonomy.

Key Fields:

Title: (text) – Name of the testbank.

Description: (text) – Overview or summary.

Creator: (User) – The educator who created it.

Creation Date: (date) – When the testbank was created.

Tags: (list of text) – Keywords, Bloom’s taxonomy levels, or custom categories.

Learning Objectives: (list of text) – Relevant objectives.

Questions: (list of Question) – Questions within this testbank.

Last Revalidated Date: (date) – Updated when questions are revalidated.

Question
Purpose: Stores individual questions with various formats and metadata.

Key Fields:

Testbank: (Testbank) – Parent testbank.

Question Text: (text) – The prompt or question content.

Question Type: (Option set) – e.g., MC, MR, CR, Hot Spot, etc.

Answer Options: (list of Answer Option) – For selectable answers.

Correct Answer(s): (list of Answer Option or text) – Correct response(s).

Difficulty Score: (number) – Value between 1–10, dynamically updated.

Creation Date: (date)

Last Validated: (date) – Updated after AI validation.

Tags & Categories: (list of text) – Additional identifiers.

AI Feedback: (text) – Summary of AI insights.

Additional Data: (JSON or text) – For question-type–specific configurations.

Answer Option
Purpose: Holds individual answer choices for questions that need them.

Key Fields:

Answer Text: (text) – Display text.

Is Correct?: (yes/no) – Indicates if it is a correct option.

Media Attachment: (file/image) – Optional media.

Display Order: (number) – For ordering (if not randomized).

3. Quality Assurance & AI Validation
Validation Log
Purpose: Records each AI-driven validation check for a question.

Key Fields:

Question: (Question) – The validated question.

Validated On: (date) – Timestamp of validation.

Validation Issues: (list of text or JSON) – Issues like typos or ambiguities.

AI Confidence Score: (number) – Optional metric from the AI.

Resolution Status: (yes/no or text) – Indicates if issues are resolved.

Comments: (text) – Additional notes.

Revision
Purpose: Maintains the change history for a question.

Key Fields:

Question: (Question) – The question being revised.

Modified By: (User) – Who made the change.

Modification Date: (date)

Change Description: (text) – Summary of the update.

Previous Data: (JSON or text) – Snapshot before change.

New Data: (JSON or text) – Snapshot after change.

Peer Review
Purpose: Collects structured feedback from colleagues.

Key Fields:

Question: (Question) – The question under review.

Reviewer: (User) – The educator providing feedback.

Comments: (text)

Rating: (number) – e.g., 1–5 scale.

Timestamp: (date)

4. Quiz/Exam Assembly & Administration
Quiz
Purpose: Represents a complete quiz or exam assembled from questions.

Key Fields:

Title: (text)

Description: (text)

Creator: (User) – The teacher who created the quiz.

Source Testbanks: (list of Testbank) – Testbanks used to source questions.

Questions: (list of Question) – Ordered or randomized list of questions.

Quiz Settings: (embedded fields or a linked data type) – e.g., time limit, shuffling options.

Schedule Start Time: (date/time)

Schedule End Time: (date/time)

Security Options:

Password Protection: (yes/no) and Password: (text)

IP Locking: (yes/no)

Adaptive Testing Rules: (list of Adaptive Testing Rule)

Proctoring Settings: (linked data type or JSON) – For proctoring integrations.

Creation Date: (date)

Quiz Settings
Purpose: Stores configuration options for the quiz.

Key Fields:

Time Limit: (number, in minutes)

Shuffle Answers: (yes/no)

Allow Multiple Attempts: (yes/no)

On-Screen Calculator: (yes/no)

Navigation Lock: (yes/no)

Feedback Configuration: (JSON or text) – For tailored feedback.

Randomization Options: (JSON/text) – Parameters for question/answer shuffling.

Adaptive Testing Rule
Purpose: Defines rules to adjust question difficulty dynamically.

Key Fields:

Rule Name: (text)

Condition: (text or JSON) – e.g., “if answered correctly.”

Action: (text) – e.g., “display a harder question.”

Parameters: (JSON/text) – Thresholds and other parameters.

5. Exam Response, Analytics & Proctoring
Quiz Result
Purpose: Stores a student’s quiz attempt along with performance data.

Key Fields:

Quiz: (Quiz) – The exam taken.

Student: (User) – The test taker.

Submission Date: (date/time)

Score: (number)

Detailed Responses: (list of Quiz Response Item)

Overall Analytics: (JSON/text) – Aggregated performance data.

Proctoring Logs: (list of Proctoring Log)

Quiz Response Item
Purpose: Details each answer provided by a student.

Key Fields:

Question: (Question)

Student Answer: (text or list of text)

Correctness: (yes/no or percentage)

Time Taken: (number, in seconds)

Feedback Provided: (text)

Proctoring Log
Purpose: Captures events flagged during an exam session.

Key Fields:

Quiz Result: (Quiz Result) – Associated exam attempt.

Event Timestamp: (date/time)

Event Description: (text)

Screenshot/Image: (image) – Optional evidence.

Flagged: (yes/no)

6. AI-Powered Educational Resources
Resource
Purpose: Stores AI-generated or curated study materials.

Key Fields:

Resource Type: (Option set) – e.g., “Study Guide”, “Lecture Notes.”

Associated With: (Testbank or Quiz) – Link to related content.

Generated Content: (rich text or file)

Media Attachments: (list of images/files)

Generation Timestamp: (date)

Improvement Plan
Purpose: Provides personalized recommendations based on exam performance.

Key Fields:

Quiz Result: (Quiz Result) – The basis for the plan.

Student: (User) – The recipient.

Recommendations: (list of text or rich text) – Topics and tips.

Additional Resources: (list of Resource)

Generated On: (date)

7. System & Notification Components
Notification/Alert
Purpose: Communicates system messages, alerts, and updates.

Key Fields:

User: (User) – The recipient.

Type: (text or Option set) – e.g., “Validation Alert”, “Exam Reminder.”

Message: (text)

Date/Time: (date/time)

Read Status: (yes/no)

Global Settings (Optional)
Purpose: Stores app-wide configuration settings.

Key Fields:

Setting Name: (text)

Setting Value: (text, number, or JSON)

Relationships & Considerations
Relational Links:

User is referenced by most data types (e.g., as Creator, Reviewer, Student).

Testbank contains a list of Questions, and each Question references its parent Testbank.

Quiz pulls in Questions either directly or from selected Testbanks.

Quiz Result ties individual student performance back to a Quiz.

Validation Logs, Revisions, and Peer Reviews are all linked to specific Questions for full audit trails.

Flexibility:

Use Option sets for roles, question types, and resource types to maintain consistency.

Use JSON/text fields (e.g., Additional Data in Question, Feedback Configuration in Quiz Settings) for dynamic or complex configurations.

Bubble’s built-in “list of” field type allows you to easily associate related items (such as lists of Questions within a Testbank).

This complete database setup covers all aspects of the Exam Generator App—from user management and content creation to quiz assembly, AI validation, exam responses, and resource generation—providing a scalable and flexible foundation for your Bubble application.
